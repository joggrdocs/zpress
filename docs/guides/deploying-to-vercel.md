# Deploy to Vercel

Deploy your zpress documentation site to Vercel as a static site.

## Prerequisites

- A [Vercel](https://vercel.com) account
- Your zpress project pushed to a Git repository (GitHub, GitLab, or Bitbucket)
- zpress building locally (`npx @zpress/kit build` succeeds)

## Steps

### 1. Add output directories to gitignore

Add zpress output directories to `.gitignore` — they are regenerated by `sync` and `build`:

```txt
.zpress/content/
.zpress/dist/
.zpress/cache/
```

### 2. Add a `vercel.json`

Create a `vercel.json` in your project root to configure the build. Commit and push this file before importing your project in Vercel.

```json
{
  "buildCommand": "npx @zpress/kit build",
  "outputDirectory": ".zpress/dist",
  "framework": null,
  "installCommand": "npm ci"
}
```

If your project uses pnpm, enable corepack in your Vercel project settings or add a `packageManager` field to your `package.json`:

```json
{
  "packageManager": "pnpm@10.32.0"
}
```

Then update `vercel.json` accordingly:

```json
{
  "buildCommand": "pnpm zpress build",
  "outputDirectory": ".zpress/dist",
  "framework": null,
  "installCommand": "pnpm install --frozen-lockfile"
}
```

### 3. Import your project in Vercel

Go to the [Vercel dashboard](https://vercel.com/new) and import your Git repository.

### 4. Configure environment variables

No environment variables are required for a standard zpress build. If your configuration fetches content from external APIs (e.g., remote OpenAPI specs), add the necessary API keys in the Vercel dashboard under **Settings > Environment Variables**.

### 5. Deploy

Push to your main branch. Vercel automatically builds and deploys on every push.

## Verification

After deployment completes, open the Vercel-provided URL. You should see your documentation site with all pages, navigation, and search working. If something looks wrong, check the build logs in the Vercel dashboard under **Deployments > (latest) > Build Logs**.

## Troubleshooting

### Build fails with missing dependencies

**Symptom:** `Module not found` errors during build.

**Fix:** Ensure your install command matches your package manager. If using pnpm, set the install command to `pnpm install --frozen-lockfile`.

### Empty site after deploy

**Symptom:** The deployed site shows a blank page or 404.

**Cause:** The output directory is misconfigured.

**Fix:** Verify the output directory is set to `.zpress/dist` (not `dist` or `.zpress`).

## Custom domain

Add a custom domain from the Vercel dashboard under **Settings > Domains**. Vercel automatically provisions SSL certificates.

Alternatively, use the Vercel CLI:

```bash
npm i -g vercel
vercel domains add docs.example.com
```

If you use a subdomain (e.g. `docs.example.com`), add a **CNAME** record pointing to `cname.vercel-dns.com` in your DNS provider. For apex domains (e.g. `example.com`), add an **A** record pointing to `76.76.21.21`.
