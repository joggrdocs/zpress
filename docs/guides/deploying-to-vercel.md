# Deploy to Vercel

Deploy your zpress documentation site to Vercel as a static site.

## Prerequisites

- A [Vercel](https://vercel.com) account
- Your zpress project pushed to a Git repository (GitHub, GitLab, or Bitbucket)
- zpress building locally (`npx @zpress/kit build` succeeds)

## Steps

### 1. Add output directories to gitignore

Add zpress output directories to `.gitignore` — they are regenerated by `sync` and `build`:

```txt
.zpress/content/
.zpress/dist/
.zpress/cache/
```

### 2. Import your project in Vercel

Go to the [Vercel dashboard](https://vercel.com/new) and import your Git repository.

### 3. Configure build settings

Set the following build settings in the Vercel project configuration:

| Setting          | Value                    |
| ---------------- | ------------------------ |
| Build command    | `npx @zpress/kit build`  |
| Output directory | `.zpress/dist`           |
| Install command  | `npm install`            |

If your project uses pnpm, enable corepack in your Vercel project settings or add a `packageManager` field to your `package.json`:

```json
{
  "packageManager": "pnpm@9.15.0"
}
```

Then set the install command to `pnpm install --frozen-lockfile` and the build command to `pnpm zpress build`.

### 4. Configure environment variables

No environment variables are required for a standard zpress build. If your config uses async content generators that call external APIs, add those API keys in the Vercel dashboard under **Settings > Environment Variables**.

### 5. Deploy

Push to your main branch. Vercel automatically builds and deploys on every push.

## Verification

After deployment completes, open the Vercel-provided URL. You should see your documentation site with all pages, navigation, and search working.

## Troubleshooting

### Build fails with missing dependencies

**Symptom:** `Module not found` errors during build.

**Fix:** Ensure your install command matches your package manager. If using pnpm, set the install command to `pnpm install --frozen-lockfile`.

### Empty site after deploy

**Symptom:** The deployed site shows a blank page or 404.

**Cause:** The output directory is misconfigured.

**Fix:** Verify the output directory is set to `.zpress/dist` (not `dist` or `.zpress`).
