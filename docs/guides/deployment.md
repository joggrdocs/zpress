# Deployment

zpress builds to static HTML — deploy it anywhere that serves static files.

## Build

Generate the production site:

```bash
npx @zpress/kit build
```

Output is written to `.zpress/dist/`. Pass `--clean` to remove previous build artifacts first:

```bash
npx @zpress/kit build --clean
```

## Preview

Preview the production build locally before deploying:

```bash
npx @zpress/kit serve
```

This starts a local static file server pointed at `.zpress/dist/`. The browser opens automatically (use `--no-open` to disable).

## Static hosting

Point your hosting provider at the `.zpress/dist/` directory. zpress produces a standard static site — no server runtime required.

| Provider     | Build command           | Output directory |
| ------------ | ----------------------- | ---------------- |
| Vercel       | `npx @zpress/kit build` | `.zpress/dist`   |
| Netlify      | `npx @zpress/kit build` | `.zpress/dist`   |
| GitHub Pages | `npx @zpress/kit build` | `.zpress/dist`   |
| Cloudflare   | `npx @zpress/kit build` | `.zpress/dist`   |

## Vercel

Add a `vercel.json` at the repo root:

```json
{
  "buildCommand": "npx @zpress/kit build",
  "outputDirectory": ".zpress/dist",
  "framework": null,
  "installCommand": "npm install",
  "ignoreCommand": "npx @zpress/kit diff"
}
```

Set `framework` to `null` — zpress handles the build itself.

The `ignoreCommand` uses `zpress diff` to skip deploys when no documentation sources changed. This reads your `zpress.config.ts` and checks all referenced files and directories against the previous commit. Exit code `0` means nothing changed (skip deploy), exit code `1` means docs changed (trigger deploy).

If you use pnpm, update the commands accordingly:

```json
{
  "buildCommand": "pnpm zpress build",
  "outputDirectory": ".zpress/dist",
  "framework": null,
  "installCommand": "pnpm install",
  "ignoreCommand": "pnpm zpress diff"
}
```

To inspect which paths zpress watches for changes:

```bash
npx @zpress/kit diff --paths
```

## GitHub Pages

Deploy with GitHub Actions using the official Pages actions.

### Workflow

Create `.github/workflows/deploy-docs.yml`:

```yaml
name: Deploy Docs
on:
  push:
    branches: [main]

permissions:
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm zpress build
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with:
          path: .zpress/dist
      - id: deployment
        uses: actions/deploy-pages@v4
```

### Repository settings

1. Go to **Settings > Pages**
2. Set **Source** to **GitHub Actions**

### Custom domain

To use a custom domain, add a `CNAME` file to `.zpress/public/`:

```txt
docs.example.com
```

Files in `.zpress/public/` are copied to the build output root.

## Joggr

:::tip Coming Soon
Joggr publishing support is not yet available. The integration is under active development.
:::

[Joggr](https://joggr.io) is a documentation platform that adds authentication and access control on top of your docs. Benefits include:

- **Authentication** — keep internal docs private with SSO and team-based access
- **Access control** — restrict visibility per section or workspace
- **Analytics** — track which docs your team reads and searches for
- **Managed hosting** — no CI/CD pipeline to maintain

Once available, publishing to Joggr will be a single command from the CLI.

## Gitignore

Add zpress output directories to `.gitignore` — they are regenerated by `sync` and `build`:

```txt
.zpress/content/
.zpress/dist/
.zpress/cache/
```
