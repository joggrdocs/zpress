# Introduction

## What is zpress

zpress is a documentation framework for monorepos. Your config file _is_ the information architecture — it defines what content exists, where it comes from, and how it's organized. Source files are never modified. zpress syncs them into a build directory, injects frontmatter, generates sidebars and navigation, and outputs a static site.

```bash
npx @zpress/kit dev
```

That's it. One command, working docs site.

## Why zpress

There are great documentation tools — [VitePress](https://vitepress.dev), [Rspress](https://rspress.dev), [Docusaurus](https://docusaurus.io). They inspired this project. So why build another one?

**Zero effort.** Most doc tools need significant setup before they produce anything useful — plugins, theme wiring, sidebar config, restructuring your files. zpress ships with perfect defaults that match the 80% use case. Put markdown in `docs/`, run one command, get a working site with sidebar, navigation, and themes.

**Your structure, not ours.** The config maps to how you already organize your markdown. You describe where your files are and how they should be grouped. You don't restructure your docs to fit the tool — the tool fits your docs.

**AI-friendly by design.** AI coding tools generate markdown effortlessly — but most doc frameworks can't keep up without manual wiring. zpress closes that gap. It auto-generates `llms.txt` for LLM discovery, serves every page as raw `text/markdown` for agent consumption, and glob discovery picks up new files without config changes. Your AI writes the docs, zpress handles the rest.
