Field & FieldGroup — Final Design

Two components, two modes: top-level grouping + expandable nesting

Example: API Response

Response
id string required
Unique identifier for the resource.
name string required
Display name of the user.
role string optional
Default: "viewer"
Access role. Controls permissions throughout the system.
metadata object optional
Arbitrary key-value metadata attached to the user.
source string optional
Where this user was imported from.
tags string[] optional
Classification tags.

Example: CLI Options

Options
--output string optional
Default: "./dist"
Output directory for the build artifacts.
--verbose boolean optional
Default: false
Enable verbose logging output.
--legacy boolean deprecated
Use the legacy build pipeline. Will be removed in v2.0.

Example: Nested Array (collapsed by default)

Returns
users User[] required
Array of user objects.
id string required
Unique identifier.
email string optional
Contact email. For deeply nested config, see UserConfig.
